请在当前论文目录中完成一次“深度论文精读”，并把最终结果写入 `summarize.md`（Markdown 格式）。

你的目标不是逐句翻译，也不是写一段泛泛摘要，而是面向中文专业读者，基于论文正文、源码、附录、图表、图片、表格、案例框、补充文件以及必要的外部资料，写出一份真正把论文讲明白的中文解读。读者默认掌握 LLM 基础知识（如 Transformer），但不预设对这篇论文的具体背景已经熟悉。

请按下面的工作原则执行：

1. 先盘点本地材料，再开始写作。
   先识别论文的主入口文件（如 `main.tex`、PDF、README、补充材料），再顺着 `\input{...}`、`\includegraphics{...}`、表格文件、附录文件把论文结构和素材摸清楚。

2. 优先使用当前目录中的本地材料。
   如果本地已经有 `figures/`、图片文件、PDF、表格文件、附录、源码或其他补充内容，直接使用这些本地文件，不要无谓重复联网下载。尤其是图片：如果论文源码中通过 `\includegraphics{...}` 引用了某张图，就把它视为必须覆盖的内容。只要本地文件可读，就直接在 `summarize.md` 中用相对路径引用，例如 `![](./figures/xxx.pdf)` 或 `![](./figures/xxx.png)`。

3. 为图表建立覆盖清单，并确保全部纳入最终文档。
   需要覆盖的对象不仅包括标准 figure/table，也包括：
   - 通过 `\input{tables/...}` 引入的表格；
   - 附录中的图和表；
   - 论文中的案例框、任务示例、提示词模板、工具列表示例、可视化对比、失败案例等承载关键信息的内容。

4. 必要时主动查证背景知识，但以论文为主。
   如果论文对某个术语、缩写、公式、训练机制、实验设定写得过于简略，允许主动联网补充背景知识、相关工作、术语定义、公开实现或作者说明；但外部资料的作用是帮助你理解和解释论文，不要把最终文档写成搜索结果拼贴。

5. 目标是“解释清楚”，不是“复述原文”。
   当论文使用高度压缩的写法、默认读者已知上下文、或省略推导细节时，请你在理解后用自然、连贯、准确的中文重新讲清楚：它为什么这样设计，这样设计解决了什么问题，和已有方法相比有什么改进，代价又是什么。

输出要求如下：

1. 所有输出内容必须全部使用中文。但对于专业术语、模块名、方法名、组件名等约定俗成的专业命名，一律直接保留英文原文，不需要额外造中文替代词；如有必要，可在首次出现时用中文补充解释其含义。

2. 最终输出文件必须是 `summarize.md`，并且内容应当是完整、可直接阅读的 Markdown 文档。

3. 文档主体应以较完整的中文段落为主，讲解要自然、连贯、有层次，像一份高质量的论文精读笔记。可以使用必要的小标题和列表，但不要把全文写成零散要点堆砌，更不要按原文逐段机械翻译。

4. 文档应系统覆盖以下内容，但不必机械套模板，结构可以自然组织：
   - 这篇论文要解决什么问题，核心挑战是什么；
   - 为什么这个问题重要，已有方法的瓶颈或不足是什么；
   - 论文的核心思想与整体方法框架；
   - 关键模块、训练流程、推理机制、算法设计、数据构造方式、奖励机制或实现技巧；
   - 重要公式及其含义；
   - 实验设置：数据集、任务定义、评价指标、对比方法、实现细节；
   - 主要实验结果、消融实验、可视化分析、案例分析、误差分析；
   - 论文结论、局限性、适用边界、尚未解决的问题；
   - 你基于论文内容形成的整体理解。

5. summarize.md 的开头格式固定为三行：
   - 第 1 行：# 中文解读标题，可由你概括拟定，要求准确、凝练，不要加类似“精读”或“总结”的词汇；
   - 第 2 行：**原文标题**
   - 第 3 行：作者列表

6. 对论文中的重要公式，请保留标准 LaTeX 形式，例如 `$...$`、`$$...$$`、`\\(...\\)`、`\\[...\\]`，并结合上下文解释：
   - 每个关键符号代表什么；
   - 公式在直观上表达了什么；
   - 它在整个方法中的作用是什么；
   - 作者为什么需要这个目标、约束或估计量。
   不要只抄公式不解释。

7. 对论文中的每一张图片，都必须在 `summarize.md` 中保留，并且尽量按论文中的出现顺序，放在对应内容附近，而不是集中堆到文末。使用 Markdown 图片语法引用本地图片文件。

8. 每张图片下都必须配有中文解读。至少说明：
   - 这张图展示了什么；
   - 它对应论文的哪一部分；
   - 图中的横轴、纵轴、模块关系、颜色、子图分工或对比对象分别意味着什么；
   - 从图中可以读出哪些关键信息；
   - 它如何支持作者的方法、实验结论或分析。
   如果一张图包含多个子图，请尽量分别解释各子图的含义及它们之间的关系。

9. 对论文中的每一张表格，也请逐一保留并解释。
   - 如果适合转成 Markdown 表格，请尽量转写；
   - 如果不适合完整转写，也要概述表格结构、关键字段、最重要的数值和对比关系；
   - 每张表都要解释：这张表想回答什么问题、实验是怎么对比的、结果说明了什么、作者希望读者从中看到什么；
   - 不要只罗列数字而不解释含义。

10. 如果论文中有案例框、模板提示词、任务示例、工具列表、定性对比、失败案例、附录图表或补充实验，也应尽量纳入正文叙述，并解释其实际意义，而不是把这些内容完全留在“附录补充”里不谈。

11. 图、表、公式、章节内容应自然融入论述过程，形成一份完整的中文解读文档，而不是“正文讲一套、图表附录另放一套”。讲到某个方法点时，就地解释相关图表；讲到某个实验结论时，就地分析相应结果。

12. 请特别避免以下问题：
   - 只做表面对照翻译，没有真正理解后的重述；
   - 对方法部分讲得过粗，只停留在口号层面；
   - 对实验部分只报数字，不解释实验设计和数字背后的含义；
   - 忽略图、表、附录、案例或可视化；
   - 详略失衡：核心方法、关键实验和重要图表讲得不够，套话却很多；
   - 未核对本地素材就断言某些图片或表格缺失。

13. 在完成 `summarize.md` 之前，请自检以下事项：
   - 是否已经覆盖论文中的全部图片；
   - 是否已经覆盖论文中的全部表格和重要案例框；
   - 是否已经解释关键公式，而不是仅仅抄写；
   - 是否已经交代实验设置、主要结果、消融或分析；
   - 图片路径是否为当前目录下可用的相对路径；
   - 文档是否整体自然流畅、像“理解后的中文解读”，而不是翻译拼接。
